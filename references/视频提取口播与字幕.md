# SOP：从视频提取口播、字幕与对话

## 需求

从本地视频中最高保真地抽取音轨，使用百炼识别口播或对话，并默认交付带时间轴的 SRT 字幕。只有用户明确要求 TXT、`.txt` 或文本文件时，才额外输出 TXT。

## 适用场景

- “把这个视频的口播文案提取出来”
- “给这个视频生成字幕”
- “识别视频里两个人的对话”
- “把视频声音转成带时间戳的文字”
- “导出视频的 SRT，需要时再给我 TXT”

## 默认交付规则

1. 默认输出 `runtime/outputs/<视频文件名>.srt`。
2. 不默认输出 TXT、Markdown 或其他纯文本副本。
3. 用户明确要求“TXT”“`.txt`”或“文本文件”时，才额外输出 UTF-8 TXT。
4. 用户只说“提取文案”但未指定格式时，仍优先交付 SRT，因为 SRT 同时保留文案与时间轴。
5. 不使用 `visual analyze-video` 识别口播；该命令只分析视频画面。
6. 不使用本地 Whisper/VAD；本地步骤只做音轨无损抽取，识别由百炼完成。

## 前置条件

- 把本地视频放入 `runtime/inputs/`。
- 安装依赖：`python -m pip install -U av click dashscope requests`。
- 已在 `scripts/.env` 配置 `DASHSCOPE_API_KEY`。
- 完整命令参数以 [CLI.md](CLI.md) 为准。

## 处理流程

```text
本地视频
  ↓ media extract-audio
原始编码可复用：数据包级复制
原始编码不兼容：无损 FLAC
  ↓ file upload
百炼签名音频 URL
  ↓ speech transcribe-long
句级与字级时间戳
  ↓
默认输出 SRT
  ↓ 用户明确要求时
额外输出 TXT
```

## 操作步骤

### Step 1：检查密钥和输入视频

```powershell
python scripts/main.py key status
Get-Item "runtime/inputs/video.mp4"
```

预期密钥状态为 `configured=true`，输入路径指向一个实际存在的视频文件。

### Step 2：最高保真抽取音轨

```powershell
$extractResult = python scripts/main.py media extract-audio "runtime/inputs/video.mp4" | ConvertFrom-Json
$audioFile = $extractResult.output_file
```

默认策略：

- AAC 输出 M4A，MP3 输出 MP3，FLAC 输出 FLAC，Opus/Vorbis 输出 OGG。
- 上述兼容编码直接复制压缩数据包，不解码、不重编码。
- 其他编码自动转为无损 FLAC。
- 保留原始采样率、声道数量和声道布局。
- 不自动降噪、降采样、转单声道或做人声增强。

输出已存在时，先确认是否允许覆盖；只有确认后才增加 `--overwrite`。

### Step 3：上传音频并取得当前 URL

```powershell
$uploadResult = python scripts/main.py file upload "$audioFile" --description "视频口播识别" | ConvertFrom-Json
$fileId = $uploadResult.uploaded_files[0].file_id
$audioUrl = $uploadResult.uploaded_files[0].url
```

长期复用时保存 `file_id`，不要长期保存签名 URL。URL 失效后执行：

```powershell
$fileResult = python scripts/main.py file get "$fileId" | ConvertFrom-Json
$audioUrl = $fileResult.data.url
```

### Step 4：按需求选择识别方式

```powershell
$srtFile = "runtime/outputs/video.srt"
```

#### 默认：普通口播、一般对话、时间戳和字幕

已知是中文时指定 `zh`；语种未知或混合语种时改用 `auto`。

```powershell
$recognitionResult = python scripts/main.py speech transcribe-long "$audioUrl" `
  --language zh `
  --itn `
  --timestamp-level word `
  --output-srt "$srtFile" | ConvertFrom-Json
$finalResult = $recognitionResult
```

该路径默认输出句级 SRT，同时在 JSON 中保留 `words[]` 字级时间戳、语言和情感字段。

#### 明确要求专业词或敏感词规则

先使用 `hotword-create` 创建热词表，再调用：

```powershell
$advancedResult = python scripts/main.py speech transcribe-advanced "$audioUrl" `
  --vocabulary-id "<热词表ID>" `
  --language-hint zh | ConvertFrom-Json
$finalResult = $advancedResult
```

根据 `transcription.transcripts[].sentences[]` 的 `begin_time`、`end_time` 和 `text` 生成 SRT。不要从控制台文本做字符串切割，应直接读取 JSON 字段。

#### 明确要求区分对话说话人

```powershell
$dialogueResult = python scripts/main.py speech transcribe-advanced "$audioUrl" `
  --diarization `
  --language-hint zh | ConvertFrom-Json
$finalResult = $dialogueResult
```

把 `transcription.transcripts[].sentences[]` 按 `begin_time` 排序后生成 SRT，每条字幕使用以下文字格式：

```text
[说话人1] 对话内容
[说话人2] 对话内容
```

`speaker_id=0` 映射为“说话人1”，`speaker_id=1` 映射为“说话人2”，依次类推。SRT 时间轴直接使用句子的 `begin_time` 和 `end_time`，单位为毫秒。

### Step 5：检查 SRT

```powershell
Get-Item "$srtFile"
Get-Content "$srtFile" -Encoding utf8
```

检查以下内容：

- 文件不为空，编码可正常显示中文。
- 字幕编号从 1 连续递增。
- 每条字幕包含 `开始时间 --> 结束时间`。
- 时间轴递增且不超过视频总时长。
- 文字没有明显漏句、重复句或专业词误识别。
- 说话人分离场景包含正确的说话人前缀。

### Step 6：仅在用户明确要求时输出 TXT

从最终识别结果中读取每条音轨的完整 `text`：

```powershell
$textParts = New-Object System.Collections.Generic.List[string]
foreach ($transcript in $finalResult.transcription.transcripts) {
  $textParts.Add($transcript.text)
}
$plainText = [System.String]::Join([System.Environment]::NewLine, $textParts)
$utf8Encoding = New-Object System.Text.UTF8Encoding($true)
[System.IO.File]::WriteAllText(
  "runtime/outputs/video.txt",
  $plainText,
  $utf8Encoding
)
```

用户没有明确要求 TXT 时，跳过本步骤。

### Step 7：处理云端文件

用户需要在后续任务中复用时保留 `file_id`。不需要复用时删除云端临时文件，释放空间和数量配额：

```powershell
python scripts/main.py file delete "$fileId"
```

默认保留本地抽取音频和 SRT，不自动删除用户本地产物。

## 预期结果

- `runtime/outputs/<视频文件名>-audio.<自动扩展名>`：最高保真抽取的音轨。
- `runtime/outputs/<视频文件名>.srt`：默认交付的字幕文件。
- `runtime/outputs/<视频文件名>.txt`：仅在用户明确要求时生成。
- 对话分离场景的 SRT 使用 `[说话人N]` 标识匿名说话人。

## 已知限制

- 背景音乐、强噪声、混响或多人重叠说话会降低识别和说话人分离准确率。
- 无损抽取只能保留源音轨质量，不能恢复视频中本来已经丢失的声音信息。
- `transcribe-advanced` 当前没有 `--output-srt` 参数；说话人分离或热词场景需要从结构化句级结果生成 SRT。
- 说话人分离只返回匿名 `speaker_id`，不能自动判断真实姓名。
- 画面中烧录的字幕不属于音轨识别范围，需要另行使用视频帧 OCR。
